Evaluación de Requisitos del Dataset

Dataset: Aforos de tráfico en la ciudad de Madrid (estaciones permanentes)

Este documento evalúa el grado de cumplimiento de los requisitos establecidos para el dataset seleccionado, que corresponde al conjunto de datos de Aforos de tráfico en la ciudad de Madrid (marzo 2025). El archivo CSV utilizado en este proyecto es DATOS_ESTACIONES_MARZO_2025.csv.

Requisitos

  • R1. Pertenencia a un dominio Smart City:
    El dataset pertenece al dominio de movilidad urbana, uno de los ejes principales de las Smart Cities. Proporciona datos sobre la intensidad del tráfico en diferentes puntos de Madrid. ✔ Cumple
  • R2. Datos disponibles en formato CSV:
    El conjunto de datos se encuentra en formato CSV, lo que facilita su análisis, transformación y conversión a RDF. El archivo subido (DATOS_ESTACIONES_MARZO_2025.csv) cumple este requisito. ✔ Cumple
  • R3. Licencia abierta:
    El dataset procede del portal de datos abiertos del Ayuntamiento de Madrid (datos.madrid.es) y está disponible bajo una licencia abierta que permite su reutilización, citando la fuente. ✔ Cumple
  • R4. Posibilidad de enlazar con entidades del mundo real:
    Cada registro incluye coordenadas geográficas (latitud y longitud) que pueden vincularse con entidades geográficas de Wikidata, Geonames o OpenStreetMap. ✔ Cumple
  • R5. Documentación del dataset (opcional):
    El portal de datos proporciona una descripción básica de los campos (intensidad, ocupación, velocidad, etc.), aunque no existe documentación técnica detallada. ⚪ Cumple parcialmente
  • R6. Datos de múltiples fuentes (opcional):
    Los datos provienen exclusivamente de sensores gestionados por el Ayuntamiento de Madrid. No se integran fuentes externas. ✖ No cumple

Conclusión

El dataset Aforos de tráfico en la ciudad de Madrid (estaciones permanentes) cumple plenamente con los requisitos obligatorios (R1–R4) y parcialmente con los opcionales (R5–R6). Es, por tanto, adecuado para su uso en la práctica de publicación de datos enlazados (Linked Data) y para desarrollar aplicaciones relacionadas con la movilidad y el tráfico urbano en Madrid.
