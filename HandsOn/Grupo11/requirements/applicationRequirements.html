Application Requirements

This document specifies the concrete requirements for the application that will be built on top of the cercan√≠as_madrid.csv dataset.

Requirements

  1. Users must be able to search stations by name, address, or code of the station.
  2. When selecting a station, it should display detailed information: station name, address, code, postal code, city, and province.
  3. It must allow filtering stations by city or province.
  4. The application should provide two views: a map view to see stations geographically, and a list view to see stations in a table/list format.

Mock-ups of the Expected User Interface

  1. Search bar and filters by city or province.
  2. Map View: a map of Madrid with markers for each station.
  3. List View: a list of stations.
  4. Station Detail View: a window showing the complete information of the selected station.
